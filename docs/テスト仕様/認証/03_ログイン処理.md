# 処理: POST /login（ログイン処理）

メールアドレスとパスワードでログインする。

## エンドポイント

| 項目 | 内容 |
|------|------|
| メソッド | POST |
| パス | /login |
| ルート名 | login |

## リクエストパラメータ

| パラメータ | 必須 | 説明 |
|-----------|------|------|
| email | ○ | メールアドレス |
| password | ○ | パスワード |
| remember | - | ログイン状態を保持 |

## テスト項目

| No | テスト項目 | 前提条件 | 操作 | 期待結果 | 優先度 |
|----|-----------|----------|------|----------|--------|
| AUTH-003 | メールアドレスでログイン成功 | 登録済みユーザー | 正しいメール・パスワードで POST | ログイン成功、/dashboard または /home へリダイレクト | 高 |
| AUTH-004 | 間違ったパスワードでログイン失敗 | 登録済みユーザー | 誤ったパスワードで POST | 認証失敗、ログイン画面に留まる | 高 |
| AUTH-LOGIN-003 | 存在しないメールアドレスでログイン | - | 未登録のメールで POST | 認証失敗、ログイン画面に留まる | 中 |
| AUTH-LOGIN-004 | バリデーションエラー（空のメール・パスワード） | - | 空パラメータで POST | 422 または リダイレクトバックでエラー表示 | 中 |

## 備考

- ログイン成功時のリダイレクト先は Jetstream 設定に依存（通常 /dashboard）
- レート制限: 5回/分（Fortify デフォルト）
