# 処理: 認証必須ページへのアクセス

認証ミドルウェアが適用されたページに未ログインでアクセスした場合の動作。

## 対象ルート（例）

| パス | ルート名 | 説明 |
|------|----------|------|
| /home | home | ホーム |
| /dashboard | dashboard | ダッシュボード |
| /settings | settings | 設定 |
| /study-progress | study-progress | 学習進捗 |
| 他、auth ミドルウェアが適用された全ルート | - | - |

## テスト項目

| No | テスト項目 | 前提条件 | 操作 | 期待結果 | 優先度 |
|----|-----------|----------|------|----------|--------|
| AUTH-007 | 未ログインで認証必須ページアクセス | - | GET /home | ログイン画面へリダイレクト | 高 |
| AUTH-007b | 未ログインで設定ページアクセス | - | GET /settings | ログイン画面へリダイレクト | 高 |
| AUTH-007c | 未ログインでダッシュボードアクセス | - | GET /dashboard | ログイン画面へリダイレクト | 高 |
| AUTH-007d | 未ログインで学習進捗ページアクセス | - | GET /study-progress | ログイン画面へリダイレクト | 中 |

## 備考

- 認証ミドルウェア: `auth:sanctum`, Jetstream の `auth_session`, `verified`
- 未認証時は `route('login')` へリダイレクト
