## 1. はじめに

### 1.1 stapla とは

stapla は**資格試験の学習計画を立て、カレンダーで管理し、学習記録をつける**ための Web アプリです。

- 資格ごとに「分野」または「サブ分野」の重みを設定すると、試験日までの学習予定が自動で生成されます。
- カレンダーで予定を確認し、その日の学習実績（勉強した分数）を記録できます。
- LINE と連携すると、朝・夜のリマインド通知を受け取れます。

### 1.2 主な用語

| 用語 | 説明 |
|------|------|
| 資格 | 受験する資格試験（例：〇〇検定）。管理画面で登録された一覧から選択します。 |
| 分野（ドメイン） | 資格を構成する大分類。計画登録時に「分野単位」で登録する場合は、この単位で重みを付けます。 |
| サブ分野（サブドメイン） | 分野をさらに細かく分けた単位。「サブ分野単位」で登録する場合に使用します。 |
| 計画 | ある資格について「いつからいつまで・どの分野に何分」と決めた学習スケジュール。1 ユーザーあたり最大 3 件まで登録できます。 |
| Todo | 計画から生成された「その日にやるべき学習」の単位。日付ごとに 1 つあり、分野ごとの予定分数が紐づきます。 |
| 学習記録 | 各 Todo に対して「実際に何分勉強したか」を入力した実績データ。 |

## 2. ログイン・アカウント

### 2.1 トップページ

- ブラウザでアプリの URL にアクセスすると、**未ログイン**の場合はログイン画面に、**ログイン済み**の場合は**ホーム**（`/home`）に遷移します。

### 2.2 通常ログイン

- メールアドレスとパスワードを入力してログインします。
- 二要素認証を有効にしている場合は、コード入力が求められます。

### 2.3 メール内ワンクリックログイン

- メールに記載されたログイン用リンクをクリックすると、パスワードなしでログインできます。
- リンクは**署名付き・有効期限 24 時間・1 回限り**です。無効なリンクの場合は 403 エラーになります。

### 2.4 ダッシュボード

- `/dashboard` にアクセスした場合：
  - **管理者**は管理画面のホームへリダイレクトされます。
  - **一般ユーザー**はホームへリダイレクトされます。

## 3. ホームとカレンダー

### 3.1 ホーム画面の構成

- **左側**: サイドバー（HOME、計画登録、学習実績、リスケジュール、設定など）
- **右側**: カレンダー（FullCalendar）

### 3.2 カレンダーの見方

- **月表示**
  - 資格ごとに色分けされた**横棒**で、学習予定日が期間として表示されます。
  - 同じ資格の連続する学習日は 1 本の棒にまとめて表示されます。
- **週表示**
  - 日付ごとに、その日の**資格名・分野（またはサブ分野）名・予定分数**の内訳が表示されます。

### 3.3 カレンダー操作

- 表示切替（月 / 週）や日付の移動は、カレンダー上部のボタンで行います。
- **日付やイベントをクリック**すると、その日の **Todo** が開き、**学習記録の入力**ができるモーダルが表示されます（後述）。

## 4. 計画登録

- サイドバーの **「計画登録」** をクリックすると、計画登録の流れが始まります。
- **計画は最大 3 件まで**登録できます。3 件登録済みの場合は、新規登録前に削除が必要です（設定画面の「計画一覧」から削除可能）。

### 4.1 登録単位の選択

最初に、次のどちらで計画を立てるか選びます。

| 選択肢 | 説明 |
|--------|------|
| **分野単位で登録する** | 資格の「分野」ごとに重みを付けて、学習日と 1 日あたりの時間を自動配分します。 |
| **サブ分野単位で登録する** | より細かい「サブ分野」ごとに重みを付けて同様に配分します。 |

### 4.2 計画登録（分野単位）の入力項目

「分野単位で登録する」を選んだ場合のモーダルで、次の項目を入力します。

| 項目 | 説明 |
|------|------|
| 勉強開始日 | 学習を開始する日（日付）。 |
| 資格名 | ドロップダウンから対象の資格を選択。 |
| 分野 | その資格の分野を選択し、それぞれに**重み**（数値）を付けます。「+」で分野を追加できます。重みの合計を基準に、各分野への学習時間が自動で配分されます。 |
| 1日の学習時間 | 1 日あたりの学習時間（分）。例：60。 |
| 余裕確保 | 計画に組み込む「予備」の割合（％）。例：10 にすると、全体の約 10％分の余裕を持った計画になります。 |
| 勉強不可日 | 勉強しない日（休みにしたい日）。日付を選んで「+」で追加し、一覧に表示されます。 |
| 受験日 | 試験の予定日（日付）。 |

入力後、**「生成」** をクリックすると計画が作成され、カレンダーに反映されます。**「キャンセル」** で閉じます。

### 4.3 計画登録（サブ分野単位）の入力項目

「サブ分野単位で登録する」を選んだ場合も流れは同様です。

- **資格名**を選ぶと、その資格に紐づく**サブ分野**が選択肢に出ます。
- 各サブ分野に**重み**を付けて、「1日の学習時間」「余裕確保」「勉強不可日」「受験日」を入力し、**「生成」** で計画を作成します。

### 4.4 重み・余裕確保の補足

- **重み**: 分野（またはサブ分野）ごとの学習量の**相対的な割合**です。全分野の重みの合計を基準に、各分野への学習時間が自動で配分されます。例：合計 3 で、ある分野の重みが 2 なら、その分野に全体の約 2/3 が割り当てられます。
- **余裕確保**: 計画にあらかじめ「予備」を入れるための設定です。10％なら全体の約 10％分の余裕を持った計画になります。

## 5. 学習記録の入力

### 5.1 学習記録モーダルを開く

- ホームの**カレンダー**で、**日付またはイベント**をクリックします。
- その日の Todo に対応する**学習記録用モーダル**が開きます。

### 5.2 表示される内容（サマリー）

- 資格名と日付が表示されます。
- その日の**分野（またはサブ分野）ごとの予定分数**が一覧で表示されます。

### 5.3 実績の入力と保存

1. **「実績入力」** をクリックすると、各項目に**実際に勉強した分数**を入力するフォームに切り替わります。
2. 各分野（サブ分野）の**実績分数**を入力します。
3. 必要に応じて**備考**（メモ）を入力できます。
4. **「保存」** をクリックすると、実績が保存されます。
   - 予定分数以上を入力した項目は「完了」として記録されます。
5. **「戻る」** でサマリー表示に戻り、**「閉じる」** でモーダルを閉じます。

## 6. リスケジュール

- 既存の計画について、**開始日・試験日・1日の学習時間・余裕確保・勉強不可日・分野（またはサブ分野）の重み**などを変更して、計画を**作り直す**機能です。
- サイドバーの **「リスケジュール」** をクリックすると、リスケジュール用モーダルが開きます。

### 6.1 操作手順

1. **対象資格**で、作り直したい計画の資格を選択します。
2. 表示される**資格名・リスケ開始日・受験日**を確認します（必要に応じてリスケ開始日は変更可能な場合があります）。
3. **1日の学習時間**「余裕確保」「勉強不可日」、および**分野またはサブ分野の重み**を必要に応じて変更します。
4. **「生成」**（または同様の実行ボタン）をクリックすると、新しい計画が作成され、以前の有効計画は無効化されます。カレンダーは新しい計画で更新されます。

※ リスケジュール可能な計画が 1 件もない場合は、「リスケジュール可能な計画がありません。」と表示されます。

## 7. 学習実績

- サイドバーの **「学習実績」** をクリックすると、学習進捗ページ（`/study-progress`）に遷移します。

### 7.1 画面の構成

- **資格選択**: ドロップダウンで、進捗を確認したい資格（目標）を選択します。
- **累積達成率・累積実績**: 選択した資格の計画に対する、累積の達成率や実績が表示されます。
- **累積進捗グラフ**: 実績累積と計画累積の推移がグラフで表示されます。
- **分野別達成率**: 分野ごとの達成度合いが表示されます。
- その他、期間を変更してデータを再取得できる場合があります。

### 7.2 対象資格がない場合

- 計画をまだ 1 件も登録していない場合などは、「対象の資格がありません」などの表示になり、グラフ等は表示されません。ホームから計画を登録してください。

## 8. 設定

- サイドバー下部の**プロフィールメニュー**から **「設定」** を選ぶと、設定ページ（`/settings`）に遷移します。

### 8.1 基本情報

- **名前**: 表示名を変更できます。編集後 **「保存」** をクリックして反映します。
- **プロフィール画像**: 「アイコン変更」で画像を選び、保存します。JPEG/PNG、5MB まで。
- **メールアドレス**: 画面には表示されますが、ここでは変更できません（無効入力の場合は別画面で変更する想定）。

### 8.2 計画一覧

- 現在登録している**計画の一覧**が表示されます。
- 各計画には**資格名・試験日・学習期間・1日あたりの学習時間**が表示されます。
- **「削除」** をクリックすると、その計画（目標資格を含む）を削除できます。削除すると元に戻せません。

### 8.3 通知設定（LINE 連携）

- **LINE連携**
  - **未連携の場合**: **「LINE連携を開始」** をクリックすると、連携用の**コード**が発行されます。表示された **QR コード**または **「友だち追加はこちら」** で LINE の友だち追加を行い、そのあと **コードを LINE にそのまま送信**すると連携が完了します。
  - **連携済みの場合**: 「LINEと連携済みです。朝・夜の通知をお届けします。」と表示されます。
- **LINE通知**: オン/オフを切り替えられます。
- **LINE通知時間**: **朝**と**夜**の時刻をそれぞれ選択できます。ここで設定した時刻に、LINE でリマインドが送られます。
- 通知設定を変更したら、**「保存」** をクリックして反映します。

### 8.4 パスワード変更

- 現在のパスワードと、新しいパスワードを入力し、**「保存」** でパスワードを変更します。

### 8.5 アカウント削除

- アカウントを削除する場合は、画面の指示に従い、確認のうえ削除を実行します。削除するとデータは復元できません。

## 9. 管理者向け機能

- 管理者としてログインしている場合、サイドバーに **「資格情報管理」**「バックアップ」「ユーザー管理」が表示されます。
- **ダッシュボード**（`/dashboard`）にアクセスすると、管理画面のホームに遷移します。

### 9.1 資格情報管理

- **資格**の一覧表示・追加・編集・削除ができます。
- 各資格に紐づく**ドメイン（分野）**・**サブドメイン（サブ分野）**の追加・編集・削除ができます。
- **CSV テンプレート**のダウンロードや、**CSV インポート**で資格・分野・サブ分野を一括登録できます。

### 9.2 バックアップ

- バックアップの**一覧表示**・**手動バックアップの取得**・**設定の更新**ができます。
- 既存のバックアップファイルの**ダウンロード**・**再試行**・**削除**ができます。

### 9.3 ユーザー管理

- 登録ユーザーの一覧表示と、各ユーザーの**編集**（有効/無効の切り替えなど）ができます。

### 9.4 学習実績（管理者）

- 管理者も **「学習実績」** から学習進捗ページにアクセスでき、進捗内容を確認できます。

## 付録：操作の流れ（早見）

| やりたいこと | 操作 |
|--------------|------|
| ログインする | トップでメール・パスワードを入力、またはメール内リンクをクリック |
| 新しい学習計画を立てる | ホーム → 計画登録 → 分野単位 or サブ分野単位を選択 → 項目入力 → 生成 |
| カレンダーで予定を確認する | ホームのカレンダーで月/週を切り替えて表示 |
| その日の勉強時間を記録する | カレンダーで日付 or イベントをクリック → 実績入力 → 分数・備考を入力 → 保存 |
| 計画をやり直したい | リスケジュール → 対象資格を選択 → 条件を変更 → 生成 |
| 進捗を確認したい | 学習実績 → 資格を選択してグラフ・達成率を確認 |
| LINE で通知を受けたい | 設定 → LINE連携を開始 → QR/友だち追加 → コードを送信 → 通知時間を設定して保存 |
| 計画を削除したい | 設定 → 計画一覧 → 対象の「削除」をクリック |
